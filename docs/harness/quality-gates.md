# Harness Hub Quality Gates

## 任务分级

### S 级：小任务

适用条件：

- 单文件或少量文档改动。
- 行为风险低，影响面清楚。
- 不需要跨模块协调。
- 可以用一个最小检查完成验证。

默认模式：

- 主代理直接完成。
- 不启用多角色。
- 最终说明包含变更和验证。

### M 级：中等任务

适用条件：

- 跨 2-4 个模块或文档。
- 需要新增或调整测试。
- 需要明确任务计划或验收记录。
- 有一定回归风险，但边界可控。

默认模式：

- 主代理负责方案和整合。
- 可按需启用勘察、验证或评审子代理。
- 建议生成 `plans/<task-slug>.md` 或 `handoffs/<task-slug>.md`。

### L 级：复杂任务

适用条件：

- 陌生大型仓库或跨前端、后端、CI、发布链路。
- 高风险行为变更。
- 多个任务点必须串行验收。
- 需要独立评审、验证或交付文档。

默认模式：

- 启用 Harness Hub 多角色模式。
- 主控负责方案、分派、复核和最终交付。
- 实现子代理负责具体 coding。
- 验证、评审和文档角色按需加入。
- 必须生成计划文件和交接文件。

## 验证矩阵

| 任务类型 | 默认检查 | 验收文档要求 |
| --- | --- | --- |
| 文档 | 标题结构、内部链接、示例命令可读性 | 说明新增/修改文档、未验证链接或外部依赖 |
| Skill | frontmatter、必需章节、引用模板、触发语 | 说明触发方式、角色流程、验证结果 |
| 代码 | 相关单元测试、格式检查、最小回归路径 | 说明测试命令、覆盖范围、未覆盖风险 |
| 前端 | 本地服务、桌面/移动截图、核心交互 | 说明浏览器、视口、交互路径和视觉风险 |
| CI | 失败日志、最小复现、修复后对应检查 | 说明失败根因、修复证据、仍可能失败的条件 |
| 发布 | 构建、迁移、配置、回滚、冒烟测试 | 发布说明、验收报告或交接文档 |

## 验收文档门槛

S 级任务：

- 最终回复中说明验证即可。
- 如果用户要求，也可以生成独立验收文档。

M 级任务：

- 至少在计划或交接文件中记录验收标准和验证结果。
- 如果涉及 PR、发布或交接，必须生成对应文档草稿。

L 级任务：

- 必须生成 `plans/<task-slug>.md`。
- 必须生成 `handoffs/<task-slug>.md`。
- 必须记录通过项、不通过项、未覆盖项和剩余风险。

## 未通过处理

验收不通过时：

- 不继续推进后续任务点。
- 回到方案、实现或测试补齐阶段。
- 重新运行相关验证。
- 在交接文档中记录失败原因和修正结果。

## 无法验证处理

无法运行检查时：

- 记录命令或检查项。
- 说明无法运行的原因。
- 给出替代检查。
- 明确剩余风险和建议的后续验证人。

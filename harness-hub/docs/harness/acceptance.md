# Harness Hub Acceptance

## 为什么验收文档重要

验收文档是工程交付的证据层。代码完成只能说明“做过”，验收文档说明“为什么这样做、如何证明有效、还有什么风险”。对 CLI coding agent 协作尤其重要，因为它让人工第二天 double check 时可以快速判断需求是否被正确理解、范围是否漂移、验证是否充分。

## 验收文档包含什么

最小验收记录应包含：

- 需求来源。
- 完成范围。
- 明确不做的范围。
- 验收标准。
- 验证命令或人工步骤。
- 验证结果。
- 未覆盖项。
- 剩余风险。
- 后续建议。

## 适用层级

S 级任务：

- 最终回复中的验证说明即可作为轻量验收。

M 级任务：

- 建议写入计划文件或交接文件。

L 级任务：

- 必须有计划文件、交接文件和验收结论。

## 人工 Double Check 清单

第二天人工复核时，优先检查：

- Spec 是否被正确理解。
- 实现范围是否超出或遗漏需求。
- 验收标准是否逐条覆盖。
- 自动检查是否真的运行并通过。
- 未覆盖风险是否可以接受。
- 交接文档是否足够让下一位工程师继续。
